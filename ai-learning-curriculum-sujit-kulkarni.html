Before You Begin

How to Use This Curriculum

This is a structured, step-by-step path. Think of it as a staircase — each phase builds on the last. You cannot skip phases and expect the later ones to make sense. Spend the time recommended, then move on.

Who is this for? Anyone — students, professionals, managers, entrepreneurs, teachers — who wants to understand and use AI meaningfully. You do not need to know how to code to complete Phases 1–4. Coding starts only in Phase 5, and even then it is gentle and guided.

Phase 1 · 2–3 Weeks

📘 AI Foundations — Understand Before You Code

Before touching any code or maths, build a solid mental model of what AI actually is, how it works at a high level, and where it is used. This phase is entirely conceptual — no prerequisites.

01
AI Foundations
⬜ Beginner · 2–3 Weeks · No Prerequisites
~15 hrs
total effort

Start here. Understand what Artificial Intelligence, Machine Learning, and Deep Learning mean — and how they differ. Learn about the real-world applications all around you. Build a vocabulary so that everything that follows makes sense.

Topics to Cover

  • What is Artificial Intelligence?
  • AI vs Machine Learning vs Deep Learning
  • Supervised, Unsupervised, Reinforcement Learning
  • How does a model "learn"? (conceptual)
  • Neural networks — intuition without maths
  • AI in everyday life — search, recommendations, ChatGPT
  • AI Ethics — bias, fairness, safety basics

Phase 2 · 4–6 Weeks

📐 Maths Essentials — Just Enough, No More

You do not need a maths degree. You need four specific topics — linear algebra, statistics, probability, and a touch of calculus — all taught visually. Skip the textbooks; use the visual resources below.

02
Maths Essentials
⬜ Beginner-friendly · 4–6 Weeks · Visual approach
~30 hrs
total effort

Focus only on what AI actually uses. Vectors and matrices (how data is represented), probability (how models express uncertainty), statistics (how we measure performance), and derivatives (how models learn via gradient descent). All taught visually — no heavy proofs.

Topics to Cover

  • Vectors, matrices, dot products
  • Matrix multiplication (how neural nets compute)
  • Mean, median, variance, standard deviation
  • Probability distributions — normal, uniform
  • Bayes theorem — conditional probability
  • What is a derivative? (gradient descent intuition)
  • Chain rule — why backpropagation works

Phase 3 · 4–6 Weeks

🐍 Python for AI — Your First Coding Language

Python is the language of AI. It is beginner-friendly, has the most AI libraries, and is what every major AI framework uses. Learn just enough Python to work with data and run models.

03
Python for AI
🟦 Beginner–Intermediate · 4–6 Weeks · First code
~35 hrs
total effort

Learn Python specifically for data science and AI — not general software development. Focus on Jupyter notebooks (the standard AI coding environment), NumPy for numbers, Pandas for data, and Matplotlib for visualisation. You will write your first working data analysis script by the end of this phase.

Topics to Cover

  • Python syntax — variables, loops, functions
  • Lists, dictionaries, comprehensions
  • Jupyter Notebook setup and workflow
  • NumPy — arrays, broadcasting, operations
  • Pandas — DataFrames, filtering, groupby
  • Matplotlib + Seaborn — charts, plots
  • File I/O — read CSV, JSON, Excel

Phase 4 · 6–8 Weeks

🤖 Machine Learning Basics — Train Your First Models

Now the real AI work begins. Use Scikit-learn — the easiest ML library — to train, evaluate, and improve real machine learning models on real datasets. This is where theory meets practice.

04
Machine Learning Basics
🟦 Intermediate · 6–8 Weeks · First real models
~50 hrs
total effort

Learn the classic ML algorithms — regression for predicting numbers, classification for predicting categories, clustering for grouping data. Learn how to evaluate whether your model is actually good. End this phase by completing a full end-to-end ML project on Kaggle.

Topics to Cover

  • Linear & Logistic Regression
  • Decision Trees & Random Forests
  • Support Vector Machines (SVM)
  • k-Nearest Neighbours (kNN)
  • k-Means Clustering
  • Train/test split, cross-validation
  • Accuracy, precision, recall, F1, AUC-ROC
  • Overfitting vs underfitting
  • Feature engineering basics

Phase 5 · 6–8 Weeks

🧠 Deep Learning — Neural Networks from Scratch

Go deeper into the architectures that power modern AI. Build neural networks, understand CNNs for images, RNNs for sequences, and the Transformer architecture that underpins GPT and BERT.

05
Deep Learning
🟦 Intermediate · 6–8 Weeks · PyTorch/TensorFlow
~55 hrs
total effort

Learn how neural networks actually work — neurons, layers, weights, activation functions, backpropagation, and gradient descent. Build CNNs for image classification, RNNs for text, and study the Transformer architecture. Use PyTorch (recommended) or TensorFlow/Keras.

Topics to Cover

  • Perceptrons, layers, activation functions
  • Backpropagation and gradient descent
  • Batch normalisation, dropout, regularisation
  • CNNs — convolution, pooling, stride
  • RNNs, LSTMs, GRUs — sequence modelling
  • Attention mechanism — the key innovation
  • Transformer architecture — how GPT works
  • Transfer learning — fine-tune pretrained models

Phase 6 · 4–6 Weeks

💬 LLMs & Generative AI — Modern AI Mastery

The cutting edge. Understand how ChatGPT, Claude, Gemini, and other large language models work. Learn prompt engineering, RAG, LangChain, and how to build real AI applications on top of these models.

06
LLMs & Generative AI
🟧 Advanced · 4–6 Weeks · Modern AI stack
~40 hrs
total effort

Learn how large language models are trained (pre-training, RLHF, fine-tuning). Master prompt engineering to get better outputs. Build a RAG application that lets an LLM answer questions about your own documents. Understand the business and ethical implications of generative AI.

Topics to Cover

  • Tokenisation — how text becomes numbers
  • Pre-training, instruction tuning, RLHF
  • Context windows and their limits
  • Prompt engineering — CoT, few-shot, roles
  • Embeddings and vector databases
  • RAG — retrieval-augmented generation
  • Fine-tuning with LoRA / PEFT
  • LangChain / LlamaIndex — build LLM apps

Phase 7 · Ongoing

🏗 Build Real Projects — Specialise & Ship

The only way to become an AI practitioner is to build things. Choose a specialisation, start competing on Kaggle, contribute to open-source, and build a public portfolio. Your projects are your CV.

07
Build Real Projects
🟧 Advanced · Ongoing · Choose your path
Ongoing
forever

Now you apply everything. Pick a specialisation track and go deep. Build 3–5 real projects, put them on GitHub, write about them. The AI field evolves monthly — staying current means reading papers, following researchers, and building constantly.

Project Ideas to Start

  • House price predictor (regression)
  • Email spam classifier (classification)
  • Movie recommendation engine
  • Image classifier (your own dataset)
  • Sentiment analysis on Twitter/Reddit
  • RAG chatbot over a PDF document
  • Kaggle competition — top 50% finish
  • Fine-tune an open-source LLM

The One Rule

The single most important principle in learning AI is this: every concept must be implemented in code. Watching videos and reading books creates the illusion of understanding. Only when you write the code, see it fail, debug it, and make it work do you actually learn. Build something every week — no matter how small. After 6 months of consistent daily effort, you will be employable. After 12 months, you will be formidable.
